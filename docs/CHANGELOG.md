---

# Version Format

Versions follow:

MAJOR.MINOR.PATCH

Example:

- 1.0.0
- 1.1.0
- 1.1.1

Where:

- **MAJOR** – Breaking changes
- **MINOR** – New features
- **PATCH** – Bug fixes

---

# Categories

Each release should be grouped under:

## Added

New functionality.

## Changed

Changes to existing functionality.

## Improved

Performance or usability improvements.

## Fixed

Bug fixes.

## Removed

Removed functionality.

## Deprecated

Features scheduled for removal.

## Security

Security improvements and fixes.

- Keep entries concise but descriptive.
- Record release dates accurately.
